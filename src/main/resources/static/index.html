Custom Remote Print Server For Oracle APEX

API que recebe o Report Query gerado pelo Oracle APEX (dados em XML) e retorna um relatório PDF customizado e dinâmico a partir de um template JasperReports (.jrxml).

Como funciona

O Oracle APEX gera Report Queries — resultados de queries convertidos para XML — e associa esses dados a um Report Layout (que define o visual final). Essa API recebe esse XML e um template JRXML, renderiza o relatório com JasperReports e retorna o PDF pronto para download ou visualização.

Oracle APEX — Instruções Rápidas
  • Registre um Remote Server no APEX WorkSpace: WorkSpace Utilities > Remote Servers com endpoint /report/generate
  • Para testes locais, use o ngrok e configure a URL: https://<ngrok-url>/report/generate
Criando a Ação Dinâmica
  1. Crie um botão na página (ex: "Gerar PDF")
  2. Adicione uma Ação Dinâmica: tipo Print Report
  3. Em Settings > Report Query, selecione a Report Query desejada
Exemplo de xmlData
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE jasperReport PUBLIC "-//JasperReports//DTD Report Design//EN"
 "http://jasperreports.sourceforge.net/dtds/jasperreport.dtd">
<jasperReport name="sample" pageWidth="595" pageHeight="842" columnWidth="555"
              leftMargin="20" rightMargin="20" topMargin="20" bottomMargin="20">
    <detail>
        <band height="20">
            <staticText>
                <reportElement x="0" y="0" width="200" height="20"/>
                <text><![CDATA[Hello Jasper!]]></text>
            </staticText>
        </band>
    </detail>
</jasperReport>
API — Info técnica

A aplicação é um serviço em Spring Boot que expõe endpoints para receber:

  • templateFile (.jrxml)
  • xmlData (XML do Report Query)
Controller / Endpoints

Rota: report/generate — multipart/form-data, retorna PDF.

Swagger

http://localhost:8080/swagger-ui/index.html

Passo a passo — Execução

Forma 1: Local (com ngrok)

mvn spring-boot:run
ngrok http 8080
https://<ngrok-url>/report/generate

Forma 2: Docker

docker build -t printserver .
docker run -p 8080:8080 printserver
